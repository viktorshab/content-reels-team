# content-reels-team — AI-команда для контент-контура

Набор AI-ассистентов для финального урока М6 курса «Бизнес на AI-скорости». Команда берёт чужие заходящие рилсы и приносит тебе **9 готовых сценариев под твою нишу + видео с говорящей головой**.

Цепочка ролей (каждая передаёт работу следующей):

```text
сбор рилсов → расшифровка → разбор «что зашло» → 9 сценариев →
проверка безопасности → сводка → говорящая голова
```

## Установка (проектно, в твой проект)

Ставь **в папку проекта**, а не глобально — это узкие роли под один контур:

```bash
npx skills add viktorshab/content-reels-team --all --agent claude-code
```

Для **Codex** замени флаг: `--agent codex`. Флаг агента обязателен — без него скиллы встанут в общий `.agents/`, и Claude Code их не увидит. С `--agent claude-code` они ставятся в `.claude/skills/` твоего проекта (локально, без `-g`).

## Состав

| Скилл | Что делает |
|---|---|
| `reels-collector` | собирает чужие рилсы, метрики, качает видео (скрипт `collect_reels.mjs` внутри) |
| `reels-transcriber` | расшифровывает видео в текст (скрипт `transcribe.mjs` внутри) |
| `content-analyst` | вытаскивает повторяемые приёмы с привязкой к числам |
| `reels-scriptwriter-brief` | сценарист: пишет 9 сценариев под твою нишу |
| `reels-scriptwriter` | генератор сценариев (его вызывает сценарист) |
| `content-safety-check` | проверка на опасные обещания (5 взглядов), не выхолащивая текст |
| `content-summary` | сводит всё в одно решение на страницу |
| `talking-head` | говорящая голова по лучшему сценарию (скрипт `make_avatar.mjs` внутри) |

## Ключи (свои, в `.env`)

Скрипты ходят в онлайн-сервисы по личным ключам из файла `.env` (в корне проекта, по умолчанию `C:/AI-projects/.env`):

- `APIFY_API_TOKEN` — сбор рилсов (apify.com)
- `ASSEMBLYAI_API_KEY` — расшифровка (assemblyai.com)
- `HEYGEN_API_KEY` — говорящая голова (developers.heygen.com); по желанию `HEYGEN_AVATAR_ID` / `HEYGEN_VOICE_ID` для своего аватара/голоса

Для сбора нужны пакеты `apify-client` и `dotenv` (`npm i apify-client dotenv`), для расшифровки — `ffmpeg`.

## Безопасность

Ключи — личные. Не выкладывай `.env` в открытый доступ, не вставляй ключи в чат. В этом наборе чужих ключей нет — каждый работает на своих.
